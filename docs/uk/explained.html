Sneekie, рядок за рядком

Українська екскурсія вихідним кодом GW-BASIC 1988 року — з поясненнями через 38 років.

Це оригінальний SNEEKIE.BAS з повернутим контекстом. Код показаний саме так, як був написаний; бурштинові нотатки під рядками і картки між секціями — це нові пояснення. Якщо процедура незрозуміла, шукайте її номер рядка нижче — усе згруповано так, як це нумерував GW-BASIC.

Одна ідея, що пояснює все: екран і є гра

Тут немає спрайтів, ігрових обʼєктів чи окремої структури даних "змія". Змія, стіни, серця і стріли — це просто символи у текстовій памʼяті екрана PC, а програма читає і записує цю памʼять напряму через PEEK і POKE.

Текстовий екран має 80×25 клітинок. Кожна клітинка має два байти: код символу, потім його колірний атрибут. Отже адреса клітинки така:

offset = (рядок − 1) × 160 + (стовпець − 1) × 2  — 160, бо рядок має 80 клітинок × 2 байти.

Парний байт містить гліф; непарний байт містить атрибут (7 = звичайний, 15 = яскравий). Щоб дізнатися, що перед змією, код просто робить PEEK байта символу в цільовій клітинці і перемикається за знайденим числом. Цей один трюк — картинка і стан є тими самими байтами — пояснює, чому так багато програми рахує екранні зміщення.

Як проходить один хід (рядки 420–1010)

  1. Чекати до Z секунд на стрілку або дозволити змії зберегти напрям.
  2. Зняти BMIN з бонусу.
  3. Перетворити напрям на крок екрана і визначити клітинку, у яку входить голова.
  4. Зробити PEEK цієї клітинки і відреагувати: порожнє місце, серце/трефа, камінь, смертельна стріла або блок стіною.
  5. Перетворити стару голову на правильний символ ═║╗╝╔╚ і намалювати нову голову █.
  6. Підсвітити тіло і просунути стріли/брами рівня на один крок.
  7. Повторювати, доки всі серця і трефи не зібрані.

Завантаження і пошук екрана

рядки 80–100
Три завдання у двох рядках. DEFINT A-Y робить змінні A…Y цілими, але залишає Z* числами з рухомою комою; тому таймер і рахунок названі Z, ZCORE, ZORE. SCREEN 0:WIDTH 80 вибирає текст 80×25, а PEEK(&H449) питає BIOS, чи це монохромний або кольоровий екран.
80 DEFINT A-Y: SCREEN 0: WIDTH 80: CLS: ZORE=0: RANDOMIZE TIMER: DEF SEG=&H0
↳ DEFINT A-Y = цілі; Z* лишаються з рухомою комою (TIMER потребує дробів, рахунок може перевищити 32767). SCREEN 0:WIDTH 80 = текст 80x25. RANDOMIZE TIMER ініціалізує генератор.
90 IF PEEK(&H449)=7 THEN VIDEO=&HB000 ELSE VIDEO=&HB800
↳ PEEK(&H449) — байт відеорежиму BIOS. Режим 7 = монохром на &HB000; інакше колір на &HB800.
100 DEF SEG=VIDEO: CLS: DIM T(15000),S(168),B(10),D(80,3)
↳ DEF SEG=VIDEO спрямовує PEEK/POKE на екранну памʼять. Масиви: T()=змія, S()=копія для вікна, B()=рядки брам, D()=стріли.

Малювання рамки і панелі рахунку

рядки 110–220
Звичайний LOCATE…PRINT малює сталі елементи: зовнішню рамку, заголовок і легенду внизу. Останній рядок робить POKE чотирьох іконок прямо у фіксовані екранні зміщення.
110 LOCATE 1, 1: PRINT CHR$(218);STRING$(78,CHR$(196));CHR$(191);
↳ Верхня межа рамки: кут + 78 ліній + кут.
120 LOCATE 2, 1: PRINT CHR$(179);SPC(78);CHR$(179);
130 LOCATE 2,17: PRINT "**** Sneekie **** (c) juli '88 by HerbySoft";
↳ Заголовок у верхній смузі.
140 LOCATE 22, 1: PRINT CHR$(179);SPC(78);CHR$(179);
150 LOCATE 22, 6: PRINT "10 punten -50 punten Highscore";
160 LOCATE 23, 1: PRINT CHR$(179);SPC(78);CHR$(179);
170 LOCATE 22,55: PRINT "Level Score";
180 LOCATE 23, 6: PRINT "25 punten Steen <ESC> vastgelopen";
190 LOCATE 23,55: PRINT "Lives Bonus";
200 LOCATE 24, 1: PRINT CHR$(192);STRING$(78,CHR$(196));CHR$(217);
↳ Нижня межа рамки.
210 POKE 3396,1: POKE 3556,10: POKE 3526,5: POKE 3366,3
↳ Покласти чотири іконки легенди просто в екранну памʼять за зміщенням: смайлик(1), камінь(10), трефа(5), серце(3).
220 '

Цикл рівнів

рядки 240–310
Кожен рівень очищає 17 внутрішніх рядків, ставить двоклітинкову змію в центрі і через ON LEVEL GOSUB переходить до рецепта цього рівня.
240 FOR LEVEL=1 TO 32
250 FOR I=1 TO 17: LOCATE 3+I,1: PRINT CHR$(179);SPC(78);CHR$(179): NEXT
↳ Очистити 17 внутрішніх рядків, лишаючи бічні стіни.
260 LOCATE 3,1: PRINT CHR$(195);STRING$(78,CHR$(196));CHR$(180)
270 LOCATE 21,1: PRINT CHR$(195);STRING$(78,CHR$(196));CHR$(180)
280 T(1)=2000: T(2)=1840: BTEL=2: ETEL=1
↳ Початкова змія: хвіст на зміщенні 2000 (рядок 13), голова на 1840 (рядок 12, прямо над ним) -> дивиться вгору. BTEL=2 — слот голови в T(), ETEL=1 — хвіст.
290 POKE T(BTEL),219: POKE T(ETEL),186: POKE T(BTEL)+1,15
↳ Малюнок: голова = блок (219), хвіст = подвійна лінія (186), голова яскрава (15).
300 E=72: F=72: HART=0: KLAVER=0: BONUS=10000: OP=0: GOSUB 1190
↳ Скидання на рівень: напрям угору (E=F=72), нічого не зібрано, бонус 10000.
310 ON LEVEL GOSUB 2320,2330,2340,2350,2360,2370,2380,2390,2410,2420,2430,2440,2450,2460,2470,2480,2320,2330,2340,2350,2360,2370,2380,2390,2410,2420,2430,2440,2450,2460,2470,2480
↳ ON LEVEL GOSUB -> рецепт рівня (2320..2480): швидкість, кількість предметів, крок бонусу і лабіринт. Рівні 17-32 повторюють ті самі 16.

Статусний рядок і розсипання предметів

рядки 320–360
Показати бонус, життя і рівень, а потім розсипати AANTAL сердець і AANTAL смайликів на випадкові порожні клітинки.
320 LOCATE 23,73: PRINT USING "######";BONUS
330 LOCATE 23,61: PRINT USING "##";LIVE: LOCATE 22,61: PRINT USING "##";LEVEL
340 FOR I=1 TO AANTAL
↳ Розсипати предмети: AANTAL разів кидаємо один смайлик (небезпека) і одне серце.
350 L=1: GOSUB 1150: L=3: GOSUB 1150: IF K1=1 THEN HART=HART+1
↳ GOSUB 1150 кладе символ у випадкову порожню клітинку. K1=1 означає, що серце лягло -> рахуємо в HART.
360 NEXT

Вікно “Рівень n”

рядки 370–410
Скопіювати частину екрана під спливним вікном у S(), намалювати рамку, дочекатися клавіші і вставити екран назад, щоб не лишити дірки.
370 FOR I=1 TO 42: FOR I3=0 TO 3: S(I+I3*42)=PEEK(1497+I+I3*160): NEXT: NEXT
↳ Зберегти ділянку 4x42 під вікном у S(), щоб потім відновити (1497 = верхній лівий кут, +160 на рядок).
380 GOSUB 2280: LOCATE 11,37: PRINT "Level ";LEVEL
390 LOCATE 12,32: PRINT "Druk op een toets"
400 DEF SEG=0: POKE 1050,PEEK(1052): DEF SEG=VIDEO: A$=INPUT$(1)
↳ Очистити буфер клавіатури BIOS (head:=tail), щоб випадкова клавіша не пропустила вікно, тоді чекати клавішу.
410 FOR I=1 TO 42: FOR I3=0 TO 3: POKE 1497+I+I3*160,S(I+I3*42): NEXT: NEXT
↳ Вставити збережену ділянку назад, стираючи вікно.

Цикл руху — читання клавіші

рядки 420–460
Серце гри. На кожному такті код чекає до Z секунд на клавішу. Ранні рівні чекають безкінечно; пізніші рухаються самі.
420 WHILE+ KLAVER+HART>0
↳ Грати рівень, поки не зібрані всі серця ТА трефи (HART+KLAVER = 0).
430 A$=INKEY$: Z2=0: Z1=TIMER
↳ Узяти клавішу, що вже чекає; запустити таймер.
440 WHILE+ A$="" AND Z2<Z
450 Z2=TIMER-Z1: A$=INKEY$
↳ Опитувати INKEY$, поки не зʼявиться клавіша або не мине Z секунд. Z=999 на початку (чекати вічно, покроково); пізніше — частка секунди (змія рухається сама).
460 WEND

Витрата бонусу і сортування клавіші

рядки 470–650
Кожен такт віднімає BMIN від бонусу. Потім код визначає, чи клавіша є стрілкою, ESC або марним вводом.
470 IF BONUS>0 THEN BONUS=BONUS-BMIN
↳ Кожен хід знімає BMIN з бонусу.
480 LOCATE 23,73: PRINT USING "######";BONUS
490 IF LEN(A$)<>1 THEN GOTO 640
↳ Справжня стрілка — рядок із 2 символів, тож довжина <> 1 означає хід (або таймаут) -> обробка на 640.
500 IF ASC(A$)<>27 THEN GOTO 910
↳ Це один символ. Якщо не ESC (27), це сміття -> 910 (малий штраф, без ходу).
510 FOR I=1 TO 3
↳ ESC (або перехід сюди після смертельного удару): процедура смерті / здачі.
520 SOUND 2000,3: SOUND 3000,3: SOUND 4000,3: SOUND 3000,3
530 NEXT
540 IF PLAY(0)<>0 THEN GOTO 540
↳ Чекати, поки спорожніє черга звуку (PLAY(0) = ноти ще лунають).
550 WHILE+ ETEL<=BTEL
↳ Розмотати всю змію, з хвоста...
560 Z=TIMER
570 Z1=TIMER-Z: IF Z1<0.075 THEN GOTO 570
580 POKE T(ETEL),32: POKE T(ETEL)+1,7: SOUND 1500,0.1
↳ ...стерти кожну клітинку (пробіл + звичайний атрибут) з писком...
590 ETEL=ETEL+1: OP=BMIN*-1: GOSUB 1190
↳ ...просуваючи хвіст і знімаючи бонус.
600 WEND
610 LIVE=LIVE-1: HART=0: KLAVER=0
↳ Втратити життя; обнулити лічильники сердець/треф.
620 IF LIVE=0 THEN LEVEL=32 ELSE LEVEL=LEVEL-1
↳ Останнє життя? LEVEL=32, щоб цикл FOR завершився (кінець гри). Інакше LEVEL-1, щоб NEXT повторив рівень.
630 GOTO 1080
640 IF LEN(A$)=2 THEN E=ASC(MID$(A$,2,1))
↳ Клавіша з 2 символів: узяти скан-код із другого байта в E (новий напрям). Таймаут лишає E без змін, тож змія рухається далі.
650 A=T(BTEL)
↳ A = де зараз голова.

Чити і поворот голови

рядки 660–700
F10 пропускає рівень, F9 додає життя і пропускає. Інакше напрям перекладається в крок екрана: ±160 для рядка, ±2 для стовпця.
660 IF E=68 THEN GOTO 1080
↳ Скан-код 68 = F10: перейти на наступний рівень.
670 IF E=67 THEN LIVE=LIVE+1: GOTO 1080
↳ Скан-код 67 = F9: безкоштовне життя і перехід далі.
680 IF E=80 THEN A=A+160 ELSE IF E=72 THEN A=A-160
↳ Перетворити напрям на крок екрана: +/-160 = рядок угору/вниз, +/-2 = стовпець ліворуч/праворуч. Тепер A — цільова клітинка.
690 IF E=77 THEN A=A+2 ELSE IF E=75 THEN A=A-2
700 D=PEEK(A)
↳ Подивитися, що вже є в тій клітинці.

Що в наступній клітинці?

рядки 710–900
Головне рішення: порожньо, трефа, серце, камінь, смайлик або стріла. Їжа збільшує змію; стріли вбивають; камені можна штовхати.
710 IF D<>32 THEN GOTO 740
↳ 32 = порожньо -> просто рухатися.
720 POKE T(ETEL),32: POKE T(ETEL)+1,7: SOUND 1500,0.1: ETEL=ETEL+1
↳ Стерти клітинку хвоста і просунути вказівник хвоста, тож змія сунеться вперед тієї ж довжини.
730 GOTO 920
740 IF D<>5 THEN GOTO 770
↳ 5 = трефа (вартість 25).
750 L=1: GOSUB 1150: GOSUB 2260
↳ Кинути десь новий смайлик (L=1) і програти дзвін — збирання ущільнює екран.
760 OP=25: GOSUB 1190: KLAVER=KLAVER-1: GOTO 920
↳ +25, на одну трефу менше, кінець. Хвіст тут не стирається, тож їжа ВИРОЩУЄ змію.
770 IF D<>3 THEN GOTO 800
↳ 3 = серце (вартість 10).
780 IF LEVEL>16 THEN L=5: GOSUB 1150: IF K1=1 THEN KLAVER=KLAVER+1
↳ На важкій половині (рівні після 16) серце також породжує нову трефу; якщо вона лягла, додати її до треф, які треба зібрати.
790 L=1: GOSUB 1150: GOSUB 2260: OP=10: GOSUB 1190: HART=HART-1: GOTO 920
↳ Породити смайлик, дзвін, +10, на одне серце менше, кінець (ріст).
800 IF D<>10 THEN GOTO 870
↳ 10 = камінь -> спробувати штовхнути.
810 TA=A: IF E=80 THEN TA=TA+160 ELSE IF E=72 THEN TA=TA-160
↳ TA = клітинка з іншого боку каменя, у тому ж напрямі.
820 IF E=77 THEN TA=TA+2 ELSE IF E=75 THEN TA=TA-2
830 D=PEEK(TA)
840 IF D<>32 THEN GOTO 910
↳ Якщо та далека клітинка не порожня, камінь не зрушити -> 910 (заблоковано).
850 POKE TA,10: POKE T(ETEL),32: POKE T(ETEL)+1,7
↳ Інакше заштовхнути камінь у TA, стерти хвіст і рухатися.
860 SOUND 1500,0.1: ETEL=ETEL+1: GOTO 920
870 IF D<>1 THEN GOTO 900
↳ 1 = смайлик — поганий.
880 FOR I=50 TO 1 STEP -1: SOUND 600+75*I,0.35: NEXT
↳ Спадний скрик із 50 нот.
890 OP=-50: GOSUB 1190: L=1: GOSUB 1150: GOTO 920
↳ -50, кинути ще один смайлик і наповзти на нього (ріст).
900 IF D=24 OR D=26 OR D=27 THEN GOTO 510
↳ 24/26/27 = рухома стріла -> миттєва смерть (перехід на 510).

Заблоковано — стіни і власний хвіст

рядок 910
